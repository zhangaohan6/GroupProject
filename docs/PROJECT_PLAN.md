## 一、这个 Project 到底要做什么

**任务**：细粒度物种识别。用 iNaturalist-2021 的子集，把生物照片分到固定的物种类别里。难点在于很多物种长得极像，而同一物种的照片在姿态、光照、背景、尺度上差异极大。

### 硬性数据要求（低于此线会「significantly reduce marks」）

| 项目 | 最低要求 | 我们的取法 |
|---|---|---|
| 类别数 | ≥ 500 种（随机采样） | 500 种，固定随机种子 |
| 每类训练图 | ≥ 50 张 | 50 张（40 训练 + 10 验证），来自 `train_mini` |
| 每类测试图 | ≥ 10 张 | 10 张，来自官方 `val` split |

关键约束：官方 test split 的标签从未公开，所以**用官方 val 当作 held-out 测试集**；train / val / test 严格隔离，绝不能数据泄漏；graded evaluation 里**不许用 iNaturalist-2021 以外的任何数据**。

### 方法要求

必须开发并测试**至少两条真正不同的完整 pipeline**：

1. **传统方法**：手工特征 + 经典分类器（如 BoVW+SIFT → SVM）
2. **深度方法**：必须**同时**包含「从随机初始化训练」和「用预训练权重」两种——这是规范明确写死的，不是二选一

### 评测指标（规范点名要求的）

top-1 准确率、top-5 准确率、overall accuracy、macro-averaged precision / recall / **F1**、混淆矩阵、以及**各方法的训练/测试耗时对比**。因为任务是细粒度多类，规范特别强调要重点看 **macro-F1 和混淆矩阵分析**，而不是只看总体准确率。

| 26–27 | 在上面基础上，**深入做 1 个** advanced 方向 |
| **28+** | **深入做 2 个以上** advanced 方向 ← **我们的目标** |

### 我们选的两个 advanced 方向

**方向 1 — Grad-CAM 可解释性**（仅深度方法）：可视化模型预测究竟由图像哪些区域驱动；检查网络关注的是生物本体还是背景；对比正确 vs 错误分类的激活图、以及易混淆物种对（例如同属的两个种）的激活图。规范特别警告：**必须用这些可视化去解释失败案例、支撑关于「模型到底学到了什么」的具体论断，而不是甩一堆热力图不做解读**。

**方向 2 — 测试时图像退化鲁棒性**：至少 4 种退化类型（高斯噪声、高斯模糊、运动模糊，可选亮度/对比度、JPEG 压缩），每种扫多个严重度等级。**只退化测试图像——不改训练数据、不重训模型**。报告 top-1 和 macro-F1 随退化类型与严重度的下降曲线，讨论哪种方法最鲁棒、以及这如何关联到各方法表征图像的方式。

> 这两个方向搭配得好：都复用同一批冻结的模型，几乎零额外训练算力，且互补（一个解释「看哪里」，一个测「看得稳不稳」）。

## 二、五人分工

核心设计思路：**先让一个人在头 3 天把「数据 + 评测」这条所有人都依赖的地基打好，之后 4 条工作线完全并行，最后靠一份统一的模型接口把两个 advanced 研究低成本地套到所有方法上。**

| 角色 | 负责人 | 主要职责 |
|---|---|---|
| **A · 数据与评测基建** | | 数据子集构建、评测框架、结果汇总、误差分析、代码仓库与 LaTeX 集成 |
| **B · 传统方法** | | 手工特征 + 经典分类器全线，纯 CPU |
| **C · 深度 · 从零训练** | | 随机初始化 CNN、训练框架、断点续训、训练动态 |
| **D · 深度 · 迁移学习** | | 预训练骨干网络微调、数据增强消融、产出**最佳模型** |
| **E · 高级研究方向** | | Grad-CAM 可解释性 + 退化鲁棒性两项研究 |

### A — 数据与评测基建（关键路径）

**第一阶段（7/20–7/22，阻塞所有人，必须最优先）**

1. 固定 500 类清单：`seed=42` 随机采样，存成 `classes_500.json` 并提交进仓库，**从此不再变动**
2. 从 42GB 的 `train_mini.tar.gz` 和 8.4GB 的 `val.tar.gz` **流式抽取**目标类别。tar.gz 是顺序流，无法随机访问，必须整条流过一遍，用目录前缀过滤；不需要落盘 42GB
3. 统一缩放到 256px 短边、JPEG 存储 → 约 **30,000 张、1.0–1.2GB**
4. **这件事全队只做一次**：产物打包上传到共享 Google Drive，其他 4 人直接挂载，不要各自重下
5. 生成 `train.csv` / `val.csv` / `test.csv` 三份 manifest（schema 见 CONTRACTS.md §2.3），冻结
6. 写共享评测模块 `evaluate.py`：吃一个 `(N, 500)` 的概率矩阵，吐出 top-1 / top-5 / overall acc / macro-P / macro-R / macro-F1 / 混淆矩阵 / 耗时

**第二阶段（7/23 起）**

- **跨方法误差分析**：全类混淆矩阵可视化 + 选定子集的细致分析 + 挖出最易混淆的物种对（这块直接对应 23–26 档的「careful error analysis」要求，也是 E 做 Grad-CAM 的输入）
- **结果汇总**：所有人的 run 都写进统一格式，A 负责汇总成报告里的所有表格
- **代码仓库 owner**（对应 Code 那 3 分）+ **CVPR LaTeX 模板 owner**

### B — 传统方法（纯 CPU，不占 GPU 额度）

- **特征**：颜色直方图/颜色矩、LBP、HOG、BoVW+SIFT
- **分类器**：Linear SVM、RBF SVM、Random Forest
- **消融**：codebook 大小（如 256/512/1024）、特征融合组合、PCA 降维维度
- 实现要点：25,000 张图 × 500 类，**用 MiniBatchKMeans 建 codebook、用 LinearSVC/SGDClassifier 而不是 RBF-SVM**，否则复杂度爆炸
- ⚠️ **预期结果很低（top-1 约 3–10%）——这是正常的**，500 类随机基线只有 0.2%。规范明说不按指标高低给分。低分本身就是报告里一个有价值的发现：手工特征在细粒度任务上会迅速触顶

### C — 深度 · 从零训练（需要 Colab Pro）

- ResNet-18 或自定义 CNN，**随机初始化**训练
- 训练框架：训练循环、**checkpoint 保存与断点续训**（Colab 会掉线，这不是可选项）、日志
- 产出**训练/验证的 loss 与 accuracy 曲线**——这是 23–26 档明确要求的「detailed reporting of training dynamics」，也是证明「模型训练充分收敛」的唯一证据
- 消融：学习率、优化器、batch size、训练轮数
- ⚠️ 20,000 张图训 500 类，从零训练会明显欠拟合/过拟合，**预期 top-1 约 5–20%**。这个「从零 vs 预训练」的巨大落差正是报告里最有说服力的对比之一

### D — 深度 · 迁移学习（需要 Colab Pro，产出最佳模型）

- **ImageNet 预训练**骨干：ResNet-50 / EfficientNet-B0 / ConvNeXt-T 或 ViT-S，至少对比 2–3 个架构
- 微调策略对比：linear probe（只训分类头）vs 全量微调 vs 部分冻结
- **数据增强消融**：baseline vs RandAugment vs Mixup/CutMix
- ⚠️ **产出全队的「最佳模型」**，冻结后交给 E 做 Grad-CAM 和鲁棒性研究——这是关键交接点，必须在 7/30 前完成
- 预期 top-1 约 45–70%

### E — 高级研究方向（Grad-CAM + 鲁棒性）

**第一阶段（7/20–7/29，此时还没有模型可用，先造工具）**

- 实现**退化库**：高斯噪声、高斯模糊、运动模糊、亮度/对比度、JPEG 压缩，每种 5 个严重度等级，参数化可复现
- 实现 **Grad-CAM**，先在现成的 ImageNet 预训练模型上验证正确性
- 两者都可以在完全不依赖队友产出的情况下先做完、先测通

**第二阶段（7/30–8/02，模型冻结后）**

- **Grad-CAM 分析**：正确 vs 错误分类的激活图对比；易混淆物种对（用 A 挖出的那些）的激活图对比；判断模型是在看生物本体还是背景。**每张图都要配解读和论断**
- **鲁棒性扫描**：把退化后的测试集喂给**所有方法**（含传统方法），画 accuracy-vs-severity 和 macroF1-vs-severity 曲线，讨论谁最鲁棒、为什么，关联到各方法的图像表征方式
- 分摊策略：E 提供退化函数，**各方法 owner 自己跑自己的模型**，E 负责汇总与解读——这样算力分散、E 的工作量也合理

## 三、让并行真正成立的接口约定

这是 5 人协作最高杠杆的技术决策，**7/20 kickoff 当天就要定死**。

**具体的格式、函数签名、schema 一律以 [CONTRACTS.md](CONTRACTS.md) 为准**，本节只说为什么它们重要。

### 1. 统一的方法接口

所有方法（传统 / 从零 / 迁移）都实现同一个抽象基类，核心是：

```python
def predict_proba(self, df, image_transform=None) -> np.ndarray:  # (N, 500)
```

**为什么这是整份契约的核心**：

- A 的 `evaluate.py` 可以对所有方法一视同仁地算全部指标
- E 的鲁棒性扫描只需要传一个 `image_transform` 进去，**完全不需要碰任何人的模型内部**

`image_transform` 这个可选参数**必须从第一天就存在**。等到 7/30 模型都冻结了再来加，意味着要回头改 B、C、D 三个人的代码——而那时只剩一周。没有这个契约，鲁棒性研究会退化成 5 个人手工对接的噩梦。

### 2. 统一的图像加载入口

所有人必须用 A 提供的 `load_image()`，不得自行 `cv2.imread` 或 `Image.open`。理由：cv2 返回 BGR、PIL 返回 RGB，混用会让 E 的同一个退化函数在不同方法上行为不一致——**这类 bug 不会报错，只会静默地让结论变错**。

### 3. 数据与结果契约

- `classes_500.json` 冻结 `class_id` 0–499 的映射，并附带分类学层级（Grad-CAM 要分析「同属的两个种」，误差分析要按 genus/family 聚合）
- 每次 run 落一个结果 JSON（全部指标 + **训练/测试耗时**）和一个 top-5 预测 npz
- **不存完整的 (5000, 500) 概率矩阵**——10MB/run 会撑爆仓库并顶到 ZIP ≤40MB 的提交上限。top-5 索引 + 概率 + 真值（约 160KB）足以复算全部要求的指标

## 六、算力建议（关于 Colab）

**建议只给 C 和 D 两人各买一个月 Colab Pro（约 A$15–19/人/月，全队约 A$30–40）**，A / B / E 用免费版足够——B 是纯 CPU，E 的工作几乎不耗 GPU，A 的数据处理也是 CPU 任务。

免费版 T4 也确实可行（规范说他们测试过），但**必须严格实现 checkpoint 断点续训**——这已经写进 C 的职责里了。

## 七、验证方式

这个计划的产出不是代码，而是一套团队执行约定。验证靠里程碑关卡：

1. **M1 验证**：`evaluate.py` 用随机预测跑一遍 test manifest，top-1 应该 ≈ 0.2%（1/500）、top-5 ≈ 1%。数字对得上，说明评测框架和数据划分都没问题
2. **数据泄漏检查**：写个脚本断言 train / val / test 三份 manifest 的文件路径集合两两交集为空；并确认 test 全部来自官方 `val` split
3. **M2 验证**：三条 pipeline 各自的 `predict_proba` 输出形状都是 `(N, 500)`、每行和为 1，且都能被同一个 `evaluate.py` 吃进去
4. **收敛验证**：C 和 D 的训练曲线必须显示 loss 已经平稳、验证准确率不再上升——这是规范反复强调的「models are sufficiently trained and converged」
5. **提交前检查**：报告用官方 CVPR 模板渲染、正文 ≤10 页、PDF ≤10MB；视频 ≤100MB 且 5 人都出镜报名；代码 ZIP ≤40MB **且不含模型权重/图像**
